## Session Start Checklist

> **Note for agent:** This checklist is not bureaucratic overhead. Each step targets a specific
> failure mode. Skipping step 4 (character profiles) produces characters who sound identical
> across scenes. Skipping step 5 (location register) produces locations that read as generic
> placeholders. Skipping step 6 (glossary) produces domain errors in period or technical scenes.
> The checklist takes 8–12 minutes and prevents revision work that takes hours.

Before each writing or revision session, complete these steps in order:

1. [ ] Read this index (2 minutes — check current phase, confirm which docs govern this session)
2. [ ] Read the relevant chapter entry in `05_narrative_architecture.md` (2 minutes — know the chapter intent before writing)
3. [ ] Read all character profiles in `02_character_registry.md` for characters appearing in this chapter (3–5 minutes)
4. [ ] Read the relevant location entry in `03_location_register.md` for settings in this chapter (1–2 minutes)
5. [ ] Read the POV section of `06_psychology_guide.md` for the chapter's POV character (2 minutes)
6. [ ] Check `04_glossary.md` for domain terms that appear in this chapter (1 minute)
7. [ ] If writing a historically-grounded scene: read the relevant verification-risk section of `07_research_foundation.md` before generating
8. [ ] If verification session: read `07_research_foundation.md` in full before starting

## Flavor Contrast: Short / Topic / Novel

> **Note for agent:** Novel-flavor has more docs and deeper docs than Short or Topic because
> a long-form dual-timeline work with 8 characters, 10 locations, and two research domains
> generates far more consistency risk across 30 chapters and 72–82 sessions. Each additional
> doc targets a specific class of drift or hallucination that would otherwise require expensive
> late-stage correction. The depth is not bureaucracy — it is technical debt prevention.
> A Short-flavor project with 20,000 words and one timeline needs none of the psychology or
> research-foundation infrastructure. A Novel-flavor project with 85,000 words and two timelines
> needs all of it, and the cost of building it is paid back by the first verification session.

| Dimension | Short-Flavor | Topic-Flavor | Novel-Flavor ([NOVEL_TITLE]) |
|-----------|-------------|-------------|-------------------------------|
| Total docs | 4–6 | 8–12 | 15–25 (this example: 8, purpose-built) |
| Character profiles | None | None | [CHARACTER_COUNT] full profiles (Doc 02) |
| Location register | None | None | [LOCATION_COUNT] locations with sensory anchors (Doc 03) |
| Glossary terms | 8–12 (shallow) | 20–28 (full definitions) | [TERM_COUNT] terms across [DOMAIN_COUNT] domains (Doc 04) |
| Chapter outline depth | 6–8 entries, 1–2 lines each | 12–14 entries, 2–3 lines each | [CHAPTER_COUNT] entries, 6 fields each (Doc 05) |
| Psychology document | None | None | Full wound/desire/fear + dynamics (Doc 06) |
| Research foundation | None | 1 partial (claims register only) | Full source inventory + risk tiers (Doc 07) |
| Sessions estimate | 15–20 | 30–50 | 65–95 (this project: [SESSION_ESTIMATE]) |
| World model build time | 2–3 sessions | 4–6 sessions | 8–10 sessions |
| Key additions vs. Topic | — | Style guide, reader profile | Character profiles, location register, psychology guide, research foundation |
