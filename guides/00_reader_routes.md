# Reader Routes

This companion uses the same route idea as the book: choose the path that matches your current depth.

## GREEN Route (Sequential Mastery)

Best for: first-time users who want structured confidence.

Estimated time to orientation: 2-3 hours.

1. Read [`../README.md`](../README.md) and [`01_first_session.md`](01_first_session.md).
2. Pick a flavor in [`../quickstart/README.md`](../quickstart/README.md).
3. Read the matching worked example index:
- Short: [`../docs/00_index.md`](../docs/00_index.md)
- Topic: [`../topic-example/00_index.md`](../topic-example/00_index.md)
- Novel: [`../novel-example/00_index.md`](../novel-example/00_index.md)
4. Copy the matching blank templates and create your own docs set.
5. Use prompts from [`../prompts/README.md`](../prompts/README.md) for generation/review.
6. Run verification before declaring progress complete.

## YELLOW Route (Practitioner Shortcut)

Best for: users who already understand prompting and want to implement fast.

Estimated time to first output: 60-120 minutes.

1. Open the matching quick-start card in [`../quickstart/`](../quickstart/README.md).
2. Copy the matching blank folder into your project.
3. Fill only the minimum starter docs for your flavor.
4. Generate one chapter and run one review pass (`AASS` + `DARE`).
5. Expand world model depth only when drift appears.

## FAST Route (Targeted Lookup)

Best for: targeted retrieval while reading or revising. In the book this is the Starter Kit lane — you are already on it.

Use this when you need one artifact immediately:

- Need chapter-to-file mapping: [`02_book_bridge.md`](02_book_bridge.md)
- Need prompt blocks: [`../prompts/README.md`](../prompts/README.md)
- Need build/tool setup: [`../tools/README.md`](../tools/README.md)
- Need flavor boundaries: [`../quickstart/README.md`](../quickstart/README.md)

## Route Switching Rule

Switch routes whenever friction changes:

- Too abstract -> move from FAST/YELLOW to GREEN for structure.
- Too slow -> move from GREEN to YELLOW for momentum.
- Need one exact answer -> move to FAST, then return to your primary route.
